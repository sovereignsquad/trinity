# Design System

## Purpose

This document defines the baseline visual and interaction system for `{trinity}`.

## Product Direction

`{trinity}` is an operator-facing local intelligence surface.

The visual system should feel:

- focused
- editorial
- high-signal
- calm under dense information

## macOS Design Principles

- prefer native density over oversized web-style cards
- prioritize timeline clarity and candidate comparison
- surface confidence, provenance, and recency explicitly
- avoid decorative chrome that competes with evidence

## Core UI Zones

1. timeline and evidence context
2. candidate comparison
3. frontier and send-ready output
4. feedback capture
5. system status and memory signals

## Visual Rules

- use typography hierarchy to separate evidence, candidate, and system metadata
- use color sparingly for state, confidence, and warnings
- make the send-ready candidate visually dominant
- never hide provenance and source context behind deep drilldowns
