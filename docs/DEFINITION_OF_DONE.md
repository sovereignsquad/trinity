# Definition Of Done

## Purpose

This document defines when work is actually done in `{trinity}`.

## A Change Is Done Only If

1. the issue acceptance checks are satisfied
2. the code is in the repository
3. the relevant tests or validation commands pass
4. the relevant SSOT docs are updated
5. `docs/STATUS.md` reflects the new reality
6. `docs/HANDOVER.md` leaves a clean resume point

## Required Evidence

For implementation work:

- code committed
- verification commands recorded

For architecture work:

- docs updated
- boundaries made explicit

For app work:

- buildable locally
- operator-visible behavior described
